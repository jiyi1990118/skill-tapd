# 需求分析模板

## 基础需求分析

```markdown
# 需求分析 - [需求名称]

## 一、需求理解

### 业务目标
- **要解决的问题**：[描述具体业务问题]
- **预期价值**：[可量化的业务价值，如提升 XX% 转化率]
- **成功指标**：[如何衡量成功]

### 用户角色
- **主要用户**：[角色1] - [需求描述]
- **次要用户**：[角色2] - [需求描述]

### 使用场景
**场景1：[场景名称]**
- 前置条件：[用户状态]
- 触发条件：[什么情况下使用]
- 操作流程：[步骤1] → [步骤2] → [步骤3]
- 预期结果：[完成后的状态]

### 边界条件
- **范围内**：[包含的功能]
- **范围外**：[不包含的功能]
- **约束条件**：[技术约束、合规要求]

---

## 二、需求拆解

### 用户故事（Story）
[Story-XXX] 作为 [角色]，我希望 [功能]，以便 [价值]

### 功能点
- [ ] 功能点1：[描述]
- [ ] 功能点2：[描述]

### 开发任务（Task）
**后端任务**：
- [Task-201] [任务名称] - [负责人] - [工期]

**前端任务**：
- [Task-301] [任务名称] - [负责人] - [工期]

**测试任务**：
- [Task-401] [任务名称] - [负责人] - [工期]

---

## 三、验收标准（Given/When/Then）

### 功能验收
```gherkin
Given [前置条件]
When [操作]
Then [预期结果]
```

### 性能验收
- 响应时间：P95 < [X]s
- 并发能力：[N] 用户同时在线

### 数据验收
- 数据完整性：[字段要求]
- 数据安全性：[加密要求]

---

## 四、依赖与风险

### 依赖项
- 依赖1：[外部系统/接口]

### 技术风险
- 风险1：[描述] - 应对方案：[方案]

### 上线计划
- 灰度策略：[比例、时间]
- 回滚预案：[触发条件、回滚步骤]
```

---

## 含原型图的需求分析

```markdown
# 需求分析 - [需求名称]

## 一、原型图解析

### 图片信息
- 图片类型：需求原型 / UI 设计稿
- 解析时间：[时间]

### UI 元素清单
从 Vision Parser 提取：
- 页面标题：[标题]
- 导航菜单：[菜单项1], [菜单项2]
- 输入框：[字段1], [字段2]
- 按钮：[按钮1], [按钮2]

### 交互流程
从原型图识别：
```
页面A → 点击按钮B → 页面C → 操作D → 结果E
```

---

## 二、需求理解

基于原型图 + 用户描述，明确：

### 业务目标
[结合原型图功能推断的业务目标]

### 使用场景
**场景1：[从原型图流程推断]**
- 用户从 [页面A] 进入
- 填写 [字段1], [字段2]
- 点击 [按钮]
- 跳转到 [页面B]

---

## 三、需求澄清

基于原型图，需要补充：

### 缺失的业务规则
- [ ] [字段1] 的校验规则？
- [ ] [按钮] 点击后的具体逻辑？
- [ ] 异常情况如何处理？

### 缺失的数据规则
- [ ] [字段] 的数据来源？
- [ ] [字段] 的数据范围？

---

## 四、验收标准

### UI 还原验收
- [ ] 页面布局与原型图一致（误差 < 5px）
- [ ] 颜色、字体与原型图一致

### 交互验收
- [ ] 交互流程与原型图一致
- [ ] 所有按钮功能正确

### 功能验收
```gherkin
Given 用户在 [页面A]
When 用户填写 [字段] 并点击 [按钮]
Then 系统显示 [结果] 并跳转到 [页面B]
```
